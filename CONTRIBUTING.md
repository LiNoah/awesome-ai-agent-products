# Contributing

Thanks for helping improve Awesome AI Agent Products.

This repository is a curated overview, not a link dump. Please prefer entries that help builders understand the agent product landscape.

## Curation Principles

- The product, project, benchmark, or report should be real and publicly accessible.
- The entry should have clear user value or clear builder value.
- Prefer maintained projects, active products, or historically important references.
- Prefer official links, GitHub repositories, documentation, demos, credible reports, or primary sources.
- Avoid pure SEO pages, thin affiliate content, low-effort wrappers, and unverified launch pages.
- Avoid adding duplicate tools unless the new entry has a distinct use case.
- Keep notes factual, concise, and neutral.

## Entry Format

Use the table format already present in the README.

For products and projects:

```md
| [Name](https://example.com) | Type | Yes/No | Best use case | Short factual note. |
```

For product ideas:

```md
| Idea name | Target user | User pain | MVP | Monetization |
```

## Pull Request Checklist

Before opening a pull request, please confirm:

- [ ] The link is official, primary, or clearly credible.
- [ ] The entry belongs in the proposed category.
- [ ] The note explains why the entry matters.
- [ ] The entry is not a duplicate of an existing item.
- [ ] The table still renders correctly in Markdown.
- [ ] The entry is not promotional spam.

## Tone

Keep descriptions short and useful. This repository should read like a practical product landscape for builders, founders, researchers, and curious users.
